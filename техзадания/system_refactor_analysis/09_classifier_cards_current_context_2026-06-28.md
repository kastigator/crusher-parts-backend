## Общая идея

Мы приводим классификатор к модели, где разные сущности открываются в разные карточки.

Главная мысль: раздел классификатора не всегда означает "модель оборудования". В дереве есть оборудование, крепеж, материалы, услуги, транспорт, расходники и другие сущности. Поэтому интерфейс карточки должен зависеть от типа ветки/раздела:

- для оборудования нужна карточка модели оборудования;
- для товара/номенклатуры нужна карточка товара;
- для папок/промежуточных разделов нужен обзор ветки;
- для услуг, материалов, транспорта и других верхних групп позже могут появиться свои карточки или варианты карточек.

Сейчас основной живой сценарий для шлифовки - карточка модели оборудования, потому что она самая сложная: паспорт, BOM, машины клиентов, клиентские исполнения.

## Важное архитектурное решение

Не надо делать одну универсальную карточку, которая пытается быть всем сразу.

Правильнее идти так:

1. Тип ветки определяет тип открываемых карточек.
2. Для каждого типа карточки свой набор вкладок.
3. Общие элементы переиспользуются, но вкладки не должны показываться там, где они не имеют смысла.

Пример:

- модель оборудования: `Паспорт`, `BOM модели`, `Машины клиентов`, `Клиентские исполнения`;
- товар/номенклатура: паспорт товара, применяемость/где используется, поставщики, документы;
- папка/обзор: список подразделов и найденных карточек внутри ветки;
- услуги/работы: отдельная карточка услуги, без BOM модели и машин клиентов.
