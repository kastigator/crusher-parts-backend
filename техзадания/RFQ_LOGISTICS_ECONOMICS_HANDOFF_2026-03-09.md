## 8. Бизнес-логика консолидации: что принять за правильную

### 8.1. Автоконсолидация

Базовый ключ группировки:

- поставщик
- страна происхождения
- `Incoterms`
- `Пункт Incoterms`

Это уже сделано и нужно сохранить.

### 8.2. Что ещё добавить

Следующий правильный уровень:

- окно готовности / ready date window
- freight mode / corridor
- при необходимости таможенный split по ТН ВЭД-группам

### 8.3. Ручная консолидация

Ручная логика должна:

- позволять перекидывать строки между группами
- не запрещать override
- но явно показывать warning-ы, если:
  - в одной группе смешаны разные `Incoterms`
  - разные `Incoterms place`
  - разные origin country
  - отсутствуют веса
  - отсутствуют данные для таможни / ТН ВЭД

То есть ручная консолидация = осознанный override, а не тихое смешивание несовместимых строк.
